1. Executive Summary

AI Talent Hub is a modern, intelligent web-based recruitment management platform designed to connect job candidates with recruiters using artificial intelligence. The platform automates the process of resume parsing and candidate screening.

By utilizing a rule-based keyword matching algorithm (AI Matching Engine), the backend assesses a candidate's skills, professional bio, work history, and academic qualifications against a recruiter's job requirements. This calculates a real-time suitability percentage. The platform also provides tools to build digital resumes, track application status, and schedule recruitment interviews.

2. Introduction

2.1 Background

In contemporary recruitment workflows, HR personnel sift through hundreds of applications daily. The manual review process is slow, error-prone, and can result in missing qualified individuals. Thus, modern software tools must assist recruiters in filtering applications and prioritizing the most matching candidates.

2.2 Problem Statement

Traditional recruitment pipelines suffer from:

  • High CV Volume: Manual evaluation of hundreds of applications per job posting is inefficient.
  • Keyword Discrepancies: Recruiters struggle to determine if a candidate's background matches technical job specs without reading full portfolios.
  • Disjointed Tracking: Candidates lack visibility into their application stages, and recruiters have to manage multiple tools for job postings, shortlisting, and technical interviews.

2.3 Objectives

  • Develop an integrated, web-based platform supporting candidate profiles and recruiter company interfaces.
  • Implement a secure role-based authentication module using JSON Web Tokens (JWT).
  • Build an automated matching module that compares job criteria and candidate data to yield a match rating.
  • Incorporate technical interview scheduling tools directly mapped to applications.

5. Database Design

5.1 Entity Relationship Diagram (Conceptual Layout)

The following entity relations are configured in AppDbContext.cs:

  • Users (1-to-1) CandidateProfiles: A registered User can have one Candidate profile.
  • Users (1-to-1) RecruiterProfiles: A registered User can have one Recruiter profile.
  • RecruiterProfiles (1-to-Many) JobPosts: A recruiter company can post multiple job vacancies.
  • JobPosts (1-to-Many) Applications: Multiple applications can be submitted for a single job posting.
  • CandidateProfiles (1-to-Many) Applications: A candidate can apply for several jobs.
  • Applications (1-to-Many) Interviews: An application can progress to have interview schedules.

6. Detailed Module Implementation

6.1 User Authentication & Security (JWT)

Secure authentication is maintained using JSON Web Token (JWT) logic. When a user requests login or registration, the backend authenticates credentials and signs a secure JWT bearer token utilizing HMAC SHA256 algorithm.

C# JWT Token Generation Implementation (Snippet from AuthController.cs):

private string GenerateJwtToken(User user)
{
    var claims = new[]
    {
        new Claim(ClaimTypes.NameIdentifier, user.Id.ToString()),
        new Claim(ClaimTypes.Email, user.Email),
        new Claim(ClaimTypes.Role, user.Role),
        new Claim("FullName", user.FullName)
    };

    var key = new SymmetricSecurityKey(Encoding.UTF8.GetBytes(_config["Jwt:Key"]!));
    var creds = new SigningCredentials(key, SecurityAlgorithms.HmacSha256);

    var token = new JwtSecurityToken(
        issuer: _config["Jwt:Issuer"],
        audience: _config["Jwt:Audience"],
        claims: claims,
        expires: DateTime.Now.AddDays(7),
        signingCredentials: creds
    );

    return new JwtSecurityTokenHandler().WriteToken(token);
}

6.3 Automated Resume Parsing & Skill Extraction

The ResumesController.cs implements an automated parser. Paste-in text resumes are scanned for high-demand developer skill tags. Match hits are automatically populated to the candidate's skills database.

Heuristic Parsing Logic (Snippet from ResumesController.cs):

var commonSkills = new List<string> {
    "C#", ".NET", "ASP.NET Core", "React", "Angular", "Vue", "JavaScript", "TypeScript", 
    "HTML", "CSS", "SQL", "Microsoft SQL Server", "SQLite", "Python", "Java", "Git", "GitHub"
};

var extractedSkills = new List<string>();
foreach (var skill in commonSkills)
{
    if (dto.ResumeText.Contains(skill, StringComparison.OrdinalIgnoreCase))
    {
        extractedSkills.Add(skill);
    }
}
// Skills are saved automatically as a semicolon-separated string
profile.Skills = string.Join(";", profile.Skills.Split(';').Union(extractedSkills).Distinct());

6.4 AI Matching Engine Algorithm

The matching percentage represents the ratio of candidate skills matching the job post specifications. The algorithm works as follows:

  1. Splits the job requirements string into list of keywords.
  2. Checks if the keywords are present in candidate's skills list.
  3. If not, it performs a secondary case-insensitive text check in the candidate's experience history JSON, bio, and educational records.
  4. Calculates the score using the formula:
Match Score % = (Count of Matched Skills / Total Job Requirements) * 100

C# Matching Logic Code (from MatchingService.cs):

double score = (double)matchedSkills.Count / requirements.Count * 100;
score = Math.Round(score, 1);

var result = new
{
    score = score,
    summary = $"Candidate matches {matchedSkills.Count} out of {requirements.Count} job requirements ({score}%).",
    matches = matchedSkills,
    missing = missingSkills,
    details = explanationNotes
};

6.5 Interview Scheduling System

Recruiters can schedule interviews directly from the job application interface. By mapping an Interview entity to the ApplicationId, recruiters record the meeting time, meeting links (e.g. Google Meet or MS Teams link), and special guidelines. The applicant is notified in real time on their candidate dashboard.

7. Deployment & Setup Guide

To run the application locally, verify the following setup stages:

Prerequisites

  • Node.js (v18 or higher) installed.
  • .NET SDK 8.0 installed.

Step 1: Setup Backend database and launch Web API

cd Backend
dotnet run

This command installs dependencies, ensures the SQLite database file aitalenthub.db is created, triggers the database seeder to populate mock profiles, and launches the ASP.NET Core server.

Step 2: Launch Frontend Server

# Open a new terminal in the project root directory
node server.js

This launches the frontend server, listening at http://localhost:3000.

9. Conclusion and Future Enhancements

AI Talent Hub successfully addresses the problems of manual candidate screening by integrating profile management, job posting, automatic resume parsing, and algorithmic matching into one seamless workflow.

Future development phases will target:

  • Semantic AI Model Matching: Integrating deep-learning models (like LLMs) to match candidates based on contextual semantics rather than simple keyword matches.
  • PDF Text Extraction: Implementing a file parsing engine in the backend controller to automatically read text from PDF documents.
  • Automated Coding Assessments: Setting up coding tests that score applicant answers automatically.
